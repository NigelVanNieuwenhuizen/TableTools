TableTools Manual

Last Updated: 2026-01-24 15:03:56

TableTools is a zero‑dependency Python library for working with tabular data in a way that is explicit, predictable, and fully under the user’s control. It provides a complete, end‑to‑end environment for reading, transforming, analyzing, and visualizing structured data — all without relying on external packages or hidden abstractions.

At the highest level, TableTools is organized around two core components:

  • The top‑level API, which manages file I/O, directory handling, workflow utilities, and access to the toolbox ecosystem.
  • The Table object, a dedicated structure for storing, cleaning, reshaping, analyzing, and visualizing tabular data.

These components work together to support a wide range of real‑world workflows. TableTools can read and write many common and specialized file formats; clean, filter, and restructure data; perform mathematical, statistical, and text‑based operations; generate synthetic datasets; visualize results using built‑in HTML/SVG plotting tools; and export final outputs in multiple formats. Its modular toolbox system extends the library into domains such as list manipulation, date handling, matrix algebra, unit conversion, and point‑based spatial operations — enabling users to build complete analysis pipelines without external dependencies.

Setting up TableTools

Download

TableTools can be downloaded from the GitHub page. Visit the TableTools' GitHub to download the .zip file.

Set up

Once downloaded, the TableTools .zip file must be unzipped. To use TableTools in a script, copy the unzipped download package to the directory where the target script is located.

Key Objects Overview

TableTools Object

The TableTools class is the top‑level API for the entire library. It provides a unified interface for reading and writing data, managing directories, coordinating workflows, and accessing the library’s toolbox ecosystem.

from TableTools.Table_Tools import TableTools

# initialize a TableTools object to access library functionality
tt = TableTools()

File Input and Output

TableTools provides a consistent interface for reading and writing a wide range of tabular, text‑based, markup‑based, and structured formats. Whether the data comes from CSV, TSV, JSON, XML, YAML, HTML tables, SQL queries, Excel files, fixed‑width text, dBase/xBase, R dput, Markdown, LaTeX, or other text formats, TableTools handles the parsing and serialization. All readers return a Table object, and all writers accept one.

Directory and File Management

TableTools manages the user’s working environment. It can:

  • set and retrieve input/output directories
  • list files and file sizes in the input directory
  • check for file or directory existence
  • create or delete files and directories
  • control overwrite behavior

Workflow Utilities

To support long‑running or multi‑step processes, TableTools includes:

  • timers and processing‑time reporting
  • progress counters and progress bars
  • manual viewing and help utilities

These tools make it easier to monitor, debug, and document data‑processing pipelines.

Access to the Toolbox Ecosystem

TableTools exposes a suite of specialized toolboxes that extend the library into domains such as:

  • list manipulation
  • mathematical and statistical operations
  • text and date processing
  • unit conversion
  • matrix algebra
  • point‑based spatial operations
  • synthetic data generation
  • standalone plotting

Table Object

The Table object is the primary data container in the library. It stores tabular data in a structured, row‑and‑column format and provides a large collection of methods for cleaning, transforming, analyzing, and visualizing that data.

Accessing Rows, Columns, and Values

Rows and columns are conventionally accessed with Table methods that accept either an integer index (for rows and columns) or a header string (for columns). Tables also support convenient indexing syntax:

row    = table[0]
column = table["column_name"]
value  = table[row_index, column_index]

This syntax is read‑only. It allows quick access to rows, columns, and individual values, but cannot be used to assign or modify data. All mutation, restructuring, and data‑editing operations are performed through Table methods, which are described in later sections of the manual.

Capabilities of the Table Object

The Table object includes a wide range of methods for modifying, restructuring, and analyzing data. Broad areas of functionality include:

  • managing Table metadata
  • data access and extraction by row and column
  • sampling and statistics by row and column
  • reshaping data
  • appending, inserting, replacing, removing, duplicating, and shifting rows and columns
  • data sorting by row or column
  • data display and inspection
  • an interactive browser‑based, spreadsheet‑style viewer
  • Table operations including joins, splitting, comparison, transposing, and pivoting
  • filtering
  • a column expression calculator with fully‑fledged syntax, row‑level functions, scalar functions, and column transformers
  • mapping and applying custom user functions to rows and columns
  • interpolation and filling
  • plotting columns

Working with point data

This example demonstrates how TableTools supports point‑based spatial analysis through the points toolbox. While TableTools is not a full GIS system, many tabular datasets contain coordinate fields, and users often need to perform spatial operations without exporting data to external software or importing additional packages. The points toolbox provides a lightweight but capable set of tools for working with coordinates directly within TableTools, including converting Tables to PointDatasets, cleaning and normalizing coordinates, computing spatial statistics, performing geometric transformations, clipping, clustering, spatial operations, spatial joins, and converting results back to Tables. This example illustrates the general workflow for using a toolbox and shows how point‑based analysis can be integrated seamlessly into broader data processing pipelines. 
# ------------------------------------------------------------
# Example 3: Working with Point Data
# ------------------------------------------------------------

# Import the TableTools class
from TableTools.Table_Tools import TableTools

# Instantiate TableTools
tt = TableTools()

# Set input directory and overwrite behavior
tt.set_in_dir("path/to/dir")
tt.set_overwrite_state(True)


# ------------------------------------------------------------
# Reading tabular datasets containing point coordinates
# ------------------------------------------------------------

# Read Tables with coordinate columns
station_table = tt.read_latex("Station_Locations")
site_table = tt.read_text("Field_Sites")


# ------------------------------------------------------------
# Converting Tables to PointDatasets
# ------------------------------------------------------------

# Convert Tables into PointDatasets by specifying:
# - a unique ID column
# - x and y coordinate columns
# This also demonstrates toolbox access via tt.points
station_points = tt.points.table_to_points(
    station_table,
    id_col="ID",
    x_col="LON",
    y_col="LAT"
)

site_points = tt.points.table_to_points(
    site_table,
    id_col="Site_ID",
    x_col="x",
    y_col="y"
)


# ------------------------------------------------------------
# Cleaning and normalizing point data
# ------------------------------------------------------------

# Remove points with missing attribute values
cleaned_sites = tt.points.filter_missing_invalid_attributes(
    points=site_points,
    to_remove=""
)

# Normalize longitudes (e.g., wrap to -180/180)
normalized_sites = tt.points.normalize_longitude(
    points=cleaned_sites,
    mode="180"
)


# ------------------------------------------------------------
# Spatial statistics
# ------------------------------------------------------------

density = tt.points.point_density(normalized_sites)
wmc = tt.points.weighted_mean_center(
    normalized_sites,
    weight_attr=["num_observations"]
)
s_d = tt.points.standard_distance(normalized_sites)


# ------------------------------------------------------------
# Coordinate format conversions
# ------------------------------------------------------------

converted_stations = tt.points.convert_dms_to_dd(station_points)


# ------------------------------------------------------------
# Spatial transformations
# ------------------------------------------------------------

translated_stations = tt.points.translate(
    converted_stations,
    x_shift=10,
    y_shift=12.5
)

transformed_sites = tt.points.affine_transformation(
    normalized_sites,
    dx=5,
    x_factor=1.0,
    angle=45,
    pivot_y=0.5,
    reflect_axis=None,
    x_shear=5.0
)

# ------------------------------------------------------------

    points=translated_stations,
    extent=clipping_extent,
    method="outside"
)


# ------------------------------------------------------------

# ------------------------------------------------------------

    points=transformed_sites,
    eps=10,
    min_pts=3,
    attributes=["x", "y", "num_observations"]
)

# Summaries of cluster counts and statistics
summary = tt.points.cluster_summary(clustered_sites)


# ------------------------------------------------------------
# Spatial join
# ------------------------------------------------------------

# Join attributes from one PointDataset to another based on proximity
joined = tt.points.spatial_join(
    points=clipped_stations,
    to_join=clustered_sites,
    columns="cluster"
)


# ------------------------------------------------------------
# Converting back to a Table
# ------------------------------------------------------------

# Convert the final PointDataset back into a Table for writing or further analysis
output = tt.points.points_to_table(joined)

Convert text values from the input list ('vals') to the specified case and return a new list.

Parameters:

vals (list): the list of values the operation will be performed on. The input list will not be altered by the operation.

result = tt.text.case(vals, mode='upper')

Find the character at the specified index ('index') of each text value in the input list ('vals') and return a new list.

Parameters:

vals (list): the list of values the operation will be performed on. The input list will not be altered by the operation.

index (integer): the index of the character to return from each text value of the input list. If 'index' is not within the valid range for a text value, a value of None will be returned for that element.

Example:

result = tt.text.char_at_index(vals, index)

result = tt.text.concatenate(vals, list_or_string)

Check whether each text value in the input list ('vals') contains the specified substring ('substr') and return a new list of booleans.
